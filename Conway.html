Simplified Version of Conway's Game of Life

Given an input file that is a grid of live and dead cells, I designed an algorithm in C that would compute the n-th generation of the cells (code available upon request), according to the rules of Conway's Game of Life:

  • If a cell is alive and has fewer than 2 live neighbors, it dies.
  • If a cell is alive and has more than 3 live neighbors, it dies.
  • If a cell is alive and has exactly 2 or 3 live neighbors, it remains alive.
  • If a cell is dead and has exactly 3 live neighbors, it becomes alive.

Above is an input file (starting generation). The algorithm uses a 3D array to alternate between the current generation and the next generation. The next generation is computed by analyzing the current generation and getting the live and dead neighbors of each cell in the grid to determine if a specific cell should be dead or alive in the next generation. It continues this process until it has reached the goal generation.

Above is the 50th generation of the input file. This solution can be verified by playing Conway's Game of Life on the official website.
